# 星载可靠推理保护机制改进筛选规划

## 1. 目标与边界

本文不把 DMR、range guard、task output guard、ILP/背包本身包装成新方法。后续只采用“真实结果达标”的改进变体：

- DMR 变体：Guard-Gated DMR，轻量 guard 触发按需冗余。
- Range guard 变体：Adaptive Quantile Range Guard，按节点选择阈值松紧。
- Task output guard 变体：Task-Utility Consistency Guard，检测输出任务效用异常。
- 优化算法变体：Confidence-Robust Bounded-Loss Memory ILP，用置信下界收益和低内存目标选择保护方案。

当前仍只完成 Windows 阶段验证，D2000/RISC-V 不作为本阶段验收条件。

## 2. 是否采用规则

所有候选统一输出到：

`code/SPINNV2/artifacts/reports/paper_assets/mechanism_improvement_screening/`

每个候选给出三类结论：

- `adopt_as_main`：达到门槛，可进入论文主方法或主实验。
- `keep_as_ablation`：有对比价值，但不足以作为主创新。
- `reject`：不进入论文主线，只记录原因。

采用门槛：

- Guard-Gated DMR：无故障平均时延相比 always-DMR 降低 `>=30%`，关键故障恢复收益保持 always-DMR 的 `>=80%`。
- Adaptive Quantile Range Guard：clean false positive `<=1%`，并且相比 minmax 在相同误报约束下关键故障检测覆盖率提升 `>=10%`。
- Task-Utility Consistency Guard：clean false positive `<=0.5%`，关键失效降低率比当前 task output guard 提升 `>=10%`，或在相近降低率下减少重执行触发率 `>=20%`。
- Confidence-Robust Bounded-Loss Memory ILP：至少一个代表预算点保持风险收益 `>=95%`，同时峰值额外内存降低 `>=25%`。

## 3. 实施路径

1. 先使用现有 fault log、range profile、runtime cost profile 和 ProtectionPlan candidate 做离线筛选。
2. 对能用现有日志验证的候选，直接输出 CSV/JSON 结果。
3. 对需要重新重放模型输出的候选，先输出 `needs_raw_replay` 状态，不把未验证结论写入论文创新。
4. 只有候选达到采用门槛，才进入 runtime/SPK 实现；否则不改 runtime。

## 4. 当前论文写法

默认主创新仍保持为：

1. 任务后果感知的运行时对象脆弱性评估方法。
2. 风险损失受限的低内存保护配置优化方法。
3. ProtectionPlan 到 SPK/Runtime/Generated C 的可执行落地闭环。

若本轮筛选中某个 primitive 变体达到 `adopt_as_main`，再把它纳入第 4 章方法或第 5 章主实验。未达标方法只作为消融、负结果或限制说明。

## 5. 当前筛选结果

已完成真实重放验证：

- Guard-Gated DMR：`keep_as_ablation`。时延下降明显，但恢复收益保持率不足，不进入主方法。
- Adaptive Quantile Range Guard：`reject`。ResNet50 top-8 节点、每节点 32 个故障事件的真实激活重放中，`q99/q99.5/q99.9` 没有相对 `minmax` 取得关键故障检测提升，不进入主线。
- Task-Utility Consistency Guard：`adopt_as_main`。YOLOv10n 10000 个 runtime fault event 重放中，普通 task output guard 的关键失效降低率为 `50.52%`，增强任务效用一致性 guard 达到 `97.21%`，clean false positive 为 `0`，可作为检测任务特色保护方法。
- Confidence-Robust Bounded-Loss Memory ILP：`adopt_as_main`。代表预算点风险收益保持约 `97.39%`，峰值额外保护内存降低 `100%`，继续作为资源约束优化主方法。
